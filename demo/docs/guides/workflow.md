## Lifecycle

Every issue follows this flow:

1. **Idea** — raw concept, needs exploration
2. **In Design** — requirements and approach being defined
3. **Backlog** — designed and ready to implement
4. **In Progress** — actively being worked on
5. **Testing** — implementation complete, verifying correctness
6. **Documentation** — updating docs to reflect the change
7. **Done** — shipped and documented

## Board View

The kanban board shows all statuses as columns. Drag cards between columns to update their status.

## Assignees

Set the `assignee` field to claim an issue. Assigned cards show a colored left border on the board so you can spot who's working on what at a glance.
